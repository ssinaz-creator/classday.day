Section 01
배경 — 왜 이 문제를
Design Question — 설계 질문
단순히 몇 퍼센트 들었는지를 보여주는 것과,
학습이 쌓이고 확장되는 감각을 주는 것은
완전히 다른 경험입니다.
배경 설명
클래스데이는 단순 강의 소비형 플랫폼이 아닌 학습의 흐름과 성취를 체감하는 경험 설계를 목표로 합니다. 이에 따라 기존의 퍼센트 기반 진도표를 넘어, 사용자가 학습의 축적과 성장 과정을 직관적으로 인지할 수 있는 구조를 탐색하였습니다. 이 문서는 그 탐색과 결론의 전 과정을 담습니다.
Problem — 01
%는 숫자일 뿐입니다
진도 72%라는 숫자는 얼마나 배웠는지는 알려줘도, 얼마나 성장했는지는 말해주지 않습니다.
Problem — 02
완료가 성취감을 만들지 않습니다
수업을 "소비"하는 구조는 학습자에게 동기를 지속시키는 경험을 제공하지 못합니다.
Problem — 03
라이브 구조는 다릅니다
VOD 기반 서비스의 게임화 방식은 라이브 수업 기반 클래스데이에 그대로 적용할 수 없습니다.
Section 02
레퍼런스 분석
스파르타코딩클럽 — 구조 분석
구조 특징
→VOD 기반 비동기 학습 구조
→개인별 진도 속도 차이가 큼
→캐릭터 및 스토리 중심 UX
→추격·경쟁 구조로 긴장감 조성
재미 요소
→"진도사우르스가 쫓아온다" — 압박 기반 동기 유발
→동료들과 함께 달리는 구조 — 속도 경쟁
→빠르게 진행할수록 성취감 상승
→속도 기반 리더보드로 경쟁 유발
분석 요약
스파르타코딩클럽의 게임화 구조는 "속도의 차이"가 존재하는 비동기 VOD 환경에서 설계된 것입니다. 진도 앞선 사람과 뒤처진 사람 사이의 긴장감이 동기 유발의 핵심 연료입니다. 이 구조는 명확하고 효과적이지만, 클래스데이의 구조와는 전제가 다릅니다.
Section 03
구조적 한계
스파르타코딩클럽 vs 클래스데이 — 구조 비교
구분 스파르타코딩클럽 ClassDay
학습 방식 VOD (비동기) 라이브 수업 기반
진도 구조 개인별 자유 진도 동일 주차 · 동기화된 진도
속도 차이 매우 큼 (핵심 재미 요소) 거의 없음
경쟁 구조 속도 경쟁 (추격/압박) 속도 기반 경쟁 적용 어려움
게임화 가능 축 얼마나 빨리 완료했는가 얼마나 깊이, 넓게 학습했는가
핵심 문제 정의
클래스데이는 스파르타처럼 "속도 기반 재미 요소"를 적용하기 어려운 구조이다.
단순 레퍼런스 차용이 아닌, 서비스 구조에 맞는 새로운 게임화 방식이 필요하다.
결과적 한계
적용 불가 요소
—"누가 더 빠른가" 라는 경쟁 구조를 만들기 어려움
—추격·압박형 스토리 적용 시 현실과 괴리 발생
—동일 주차 구조로 인해 진도 기반 게임화 한계 존재
남은 가능성
+동일 진도 내에서도 완성도의 차이는 존재
+학습 요소 간 연결·확장은 측정 가능
+깊이와 밀도를 시각화하는 방법은 유효
Section 04
방향 전환

학습은 선형적으로 소비되는 것이 아니라
연결되고 확장되며 구조화된다.

경쟁의 기준 재정의
기존 프레임 — 속도 경쟁
—누가 더 빨리 완료했는가
—진도 퍼센트 수치 비교
—완료/미완료 이분법적 상태
—라이브 구조에서 유효하지 않음
새로운 프레임 — 확장 경쟁
→누가 더 깊이 학습했는가 — 완성도
→누가 더 많이 연결했는가 — 확장
→영역의 크기와 밀도로 성취 표현
→동일 주차 내에서도 차별화 가능
설계 방향 — 연결과 확장 중심 구조
Concept — 01
노드 연결
개별 학습 요소(노드)를 연결하며 학습 구조를 형성합니다. 연결 행위 자체가 성취 경험이 됩니다.
Concept — 02
영역 생성
연결이 닫히면 영역이 만들어집니다. 완료가 아닌 "채워짐"으로 성취를 인식합니다.
Concept — 03
확장과 밀도
영역이 넓어지고 밀도가 높아질수록 학습의 깊이가 시각적으로 드러납니다.
기대 효과
동일 진도 내에서도 개인별 학습 차이를 표현할 수 있으며, 성취를 "완료"가 아닌 "확장"으로 인식하게 합니다. 경쟁 구조는 영역의 크기와 밀도를 통해 자연스럽게 유도되며, 압박이 아닌 탐구 중심의 동기 유발이 가능합니다.
Section 05
설계 시도 1 · 2
  • V.01
    1차 시도 — Attempt
    별자리 기반 연결 구조
    학습 노드를 점으로 표현하고, 연결하면 별자리 형태가 만들어지는 구조입니다. 감성적 표현에서는 우수했으나, 진도와 성취가 직관적으로 드러나지 않는 한계가 있었습니다. 아름다운 비유였지만, 경쟁 요소와 정량적 성취 표현이 부족했습니다.
    장점 — 감성적 표현 우수 한계 — 직관성 부족 한계 — 경쟁 요소 약함
  • V.02
    2차 발전 — Territory Model
    영역 확장 구조 — 점 → 선 → 면
    노드 연결이 선이 되고, 선이 닫히면 면(영역)이 채워지는 구조입니다. 완성도에 따라 영역의 밀도가 변화하며, 성취감이 시각적으로 강화됩니다. 게임적 요소가 자연스럽게 도입되고, 동일 주차 내에서도 개인 차이를 표현할 수 있었습니다.
    해결 — 성취감 강화 해결 — 경쟁 요소 도입 발전 — 밀도 기반 시각화
새로운 문제 발견 — 다중 강좌 환경
클래스데이는 복수 강좌 수강 구조입니다. 강좌별로 영역 맵이 분리되면 전체 진도 파악이 어려워지고, 강좌 간 비교 기준도 부족해집니다.
Section 06
최종 구조 — 이중 레이어
Upper Layer
상위 레벨 — 요약 인지
빠른 상태 파악 / 비교 기준 제공
전체 학습 상태를 한눈에 파악할 수 있도록 강좌별 간단 지표를 제공합니다.
  • →강좌별 도트 인디케이터 (●●●○○)
  • →퍼센트 수치 (보조 지표)
  • →전체 학습 방향성 제공
  • →강좌 간 비교 기준
Lower Layer
하위 레벨 — 몰입 경험
성취감 강화 / 깊이 시각화
개별 강좌 진입 시 영역 확장 Canvas를 제공합니다. 학습의 연결과 확장이 시각적으로 체감됩니다.
  • →Territory Model Canvas
  • →노드 연결 → 영역 생성 인터랙션
  • →완성도에 따른 밀도 변화
  • →강좌 내 성취 몰입 경험
전체 진도율 정의
전체 진도율은 단순 강좌별 평균이 아닌, 학습 요소 기반 가중치 계산으로 산출합니다. 이 수치는 메인 UX 요소가 아닌 보조 지표로 활용됩니다. 방향성 제공과 강좌 간 비교 기준의 역할을 담당하며, 성취 경험의 주인공은 하위 레이어의 확장 Canvas입니다.
이중 레이어 역할 요약
구성 요소 역할 UX 목적
전체 진도율 학습 방향 제시 — 보조 지표 전체 상태 인지, 동기 유지
강좌 리스트 상태 인지 + 비교 기준 빠른 스캔, 우선순위 파악
영역 Canvas 몰입 + 성취 시각화 — 메인 경험 게임화, 확장감, 깊이 체감

Section 07
결론
  • C.01
    스파르타코딩클럽
    "속도 경쟁 기반 게임화"에 최적화된 구조
    VOD 비동기 환경에서 개인별 속도 차이를 활용한 게임화는 효과적으로 작동합니다. 하지만 이 구조는 속도의 차이가 전제될 때만 유효합니다.
  • C.02
    ClassDay
    동일 진도 기반 구조로 동일 접근 불가
    라이브 수업 기반의 동기화된 진도 구조는 속도 경쟁이 성립하지 않습니다. 클래스데이에는 서비스 구조에 맞는 고유한 게임화 전략이 필요했습니다.
  • C.03
    우리의 선택
    속도가 아닌, 연결과 확장
    클래스데이는 "쫓아가는 학습"이 아닌 "스스로 확장해가는 학습"을 만듭니다. 이중 레이어 구조를 통해, 빠른 상태 인지와 깊은 성취 경험을 동시에 제공합니다.
핵심 한 줄 — Design Principle
클래스데이는 쫓아가는 학습이 아니라,
스스로 확장해가는 학습을 만듭니다.
진도를 보여주는 것이 아니라, 성장을 느끼게 한다.
